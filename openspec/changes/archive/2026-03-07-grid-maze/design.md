## Context

The current map is a layered directed graph generated by `mapGen.ts` — 7 layers with 1-3 rooms per layer, connected forward-only. The player picks a path at each fork and cannot backtrack. This produces a linear experience that doesn't feel like exploring a maze.

The map UI in `RunScreen.tsx` renders these nodes as clickable buttons. The `Room` type has `connections: string[]` for forward edges and `visited: boolean`. `MapGraph` holds `rooms`, `startRoomId`, `currentRoomId`, `bossRoomId`.

## Goals / Non-Goals

**Goals:**
- Replace the layered graph with a 2D tile grid that feels like an actual maze
- Support free movement in 4 directions (up/down/left/right) through corridor tiles
- Support backtracking — cleared rooms are free to walk through
- Fog of war — only reveal tiles adjacent to visited tiles
- Place room types (Combat, Rest, Shop, Event, Boss) at meaningful positions in the maze
- Grid renders cleanly on mobile (small tiles, tap to move)
- Boss is always reachable; some dead ends contain optional high-value rooms

**Non-Goals:**
- Real-time movement or animation (still turn-based room-to-room)
- Procedural corridor rendering or wall textures (tiles are abstract squares)
- Multi-floor mazes or vertical movement
- Changing room type behavior (Combat, Rest, Shop, Event, Boss stay the same)
- Sector 2/3 content (just the infrastructure and Sector 1 maze shape)

## Decisions

### Grid size: 7x7

A 7x7 grid gives 49 cells. With ~40% walls, that's ~29 walkable tiles. Not all are rooms with encounters — many are empty corridors. Target: 8-10 encounter rooms (Combat), 1-2 Rest, 1 Shop, 1-2 Event, 1 Boss. This matches the current ~12 meaningful rooms per sector.

**Why not 5x5?** Too cramped — dead ends wouldn't feel meaningful, and the maze would be trivially small.
**Why not 9x9?** Too many rooms inflates the economy. The player could grind too many combats.

### Maze generation: Recursive backtracker with post-processing

1. Start with a full 7x7 grid of walls
2. Use recursive backtracker (DFS) to carve corridors — this naturally produces dead ends and winding paths
3. Post-process: place the Boss at the cell farthest from start (by path distance)
4. Place Rest/Shop rooms at branch junctions (accessible but not on the critical path)
5. Place Combat rooms at remaining corridor cells
6. Place Event rooms at dead ends (reward for exploration)
7. Start position: top-left area; Boss: bottom-right area (general tendency, not fixed)

**Why recursive backtracker?** It produces mazes with long corridors and clear dead ends — exactly the feel we want. It's simple to implement and always produces a perfect maze (every cell reachable).

### Room data model: Grid coordinates replace connections

```
Room {
  id: string
  type: RoomType | 'Empty'  // Empty = cleared combat or corridor
  sector: 1 | 2 | 3
  visited: boolean
  cleared: boolean          // combat completed or event resolved
  x: number
  y: number
}

GridMaze {
  grid: (Room | null)[][]   // null = wall, Room = walkable
  startX: number
  startY: number
  playerX: number
  playerY: number
  bossX: number
  bossY: number
  sector: 1 | 2 | 3
}
```

`MapGraph` is fully replaced by `GridMaze`. The `connections` field is gone — adjacency is implicit from grid coordinates (up/down/left/right neighbors that aren't null).

### Movement and backtracking

- Player can move to any adjacent (non-null, non-wall) tile
- Moving to an unvisited tile reveals it and its neighbors (fog of war)
- Moving to a Combat room that isn't cleared triggers combat
- Moving to a cleared room is free (no encounter)
- Rest/Shop/Event rooms trigger their UI on first visit; revisiting does nothing (already cleared)

### Fog of war

- A tile is "revealed" if it's adjacent (including diagonals) to any visited tile
- Revealed but unvisited tiles show their room type icon but are dimmed
- Unrevealed tiles show as dark/unknown
- The player's current position is highlighted

### Map UI

- Render as a CSS grid of small square tiles (~48px on mobile)
- Each tile shows an icon for its room type (or dark for unrevealed)
- Visited tiles are fully lit; revealed tiles are dimmed; unrevealed tiles are dark
- Player taps an adjacent walkable tile to move
- The grid auto-scrolls/centers on the player position if it doesn't fit on screen

## Risks / Trade-offs

- **[Economy inflation]** More rooms reachable via backtracking → more rewards. Mitigation: keep total encounter rooms at 8-10 via grid density. HP cost of exploration is a natural limiter.
- **[Guaranteed Rest/Shop]** Player can always backtrack to find Rest/Shop. Mitigation: this is intentional — removes bad-luck frustration. Cost is HP spent getting there.
- **[Breaking save compatibility]** MapGraph → GridMaze changes the run state shape. Mitigation: any active run in old format is abandoned (run state resets on format change). This is acceptable since runs are short.
- **[Mobile rendering]** 7x7 grid at 48px = 336px wide, fits on most phones. If not, allow pan/scroll.
